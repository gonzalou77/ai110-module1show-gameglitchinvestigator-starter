LOGIC_UTILS.PY — EXPANDED DOCSTRINGS
--------------------------------------

All four functions received expanded docstrings explaining inputs, outputs,
and behaviour at a high level.

get_range_for_difficulty:

  """
  Return the (low, high) inclusive number range for a given difficulty.

  Easy   -> (1, 20):  small range, easiest to guess
  Normal -> (1, 100): standard range
  Hard   -> (1, 200): widest range, hardest to guess
  Any unrecognised difficulty falls back to the Normal range (1, 100).
  """

parse_guess:

  """
  Validate and convert raw text input from the user into an integer guess.

  Accepts whole numbers and decimals (truncated to int, e.g. "7.9" -> 7).
  Rejects empty input, None, and non-numeric strings.

  Returns a three-element tuple:
    - ok (bool):              True if parsing succeeded, False otherwise
    - guess_int (int | None): the parsed integer, or None on failure
    - error_msg (str | None): human-readable error string, or None on success
  """

check_guess:

  """
  Compare the player's guess against the secret number.

  Returns a two-element tuple:
    - outcome (str): one of "Win", "Too High", or "Too Low"
    - message (str): an emoji-prefixed hint shown to the player

  "Too High" means the guess was above the secret -> player should go lower.
  "Too Low"  means the guess was below the secret -> player should go higher.
  """

update_score:

  """
  Calculate and return the new score after a guess.

  Win:
    Awards 100 - 10 * (attempt_number + 1) points, minimum 10.
    Fewer attempts used means a higher score.

  Too High / Too Low:
    Deducts 5 points. Wrong guesses never award points.
    Score can go negative if the player guesses poorly enough.

  Any unrecognised outcome leaves the score unchanged.
  """


APP.PY — ADDED BLOCK COMMENTS
-------------------------------

Each logical section of the script received a high-level comment explaining
its purpose and behaviour.

ATTEMPT_LIMIT_MAP constant:
  # Maps each difficulty level to the maximum number of guesses allowed.

Page config block:
  # Sets the browser tab title and favicon for the Streamlit app.

Sidebar settings block:
  # Lets the player choose a difficulty before or between games.
  # Derives the attempt limit and number range from the chosen difficulty.

Session state initialization block:
  # Streamlit reruns the entire script on every interaction, so session_state
  # is used to persist game data (secret number, attempt count, score, etc.)
  # across reruns. Each key is only initialised once per browser session.

Game UI block:
  # Displays the current range, attempts remaining, and a debug expander.
  # Renders the guess input and the three action controls (submit, new game,
  # hint toggle) as side-by-side columns.

New game handler block:
  # Resets all session state to starting values and picks a new secret number
  # within the range for the currently selected difficulty, then reruns the app.

Game-over guard block:
  # If the game has already ended (won or lost), show the appropriate message
  # and halt further script execution so the guess form is not rendered.

Guess submission handler block:
  # Increments the attempt counter, validates the input via parse_guess, then
  # checks the guess and updates the score. Ends the game when the player wins
  # or exhausts their allowed attempts.
